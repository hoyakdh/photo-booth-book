# Book Photo Booth v2 - 추가 기능 구현 계획서

## 1. 촬영 날짜 워터마크

### 목적
촬영된 사진에 날짜와 학교명을 자동으로 삽입하여 기념사진 느낌 강화

### 구현 방식
- 캡처 시 Canvas에 텍스트를 오버레이하여 PNG에 포함
- 관리자 설정에서 워터마크 내용/위치/스타일 설정 가능

### 설정 항목
```typescript
interface WatermarkConfig {
  enabled: boolean;
  text: string;        // 예: "○○초등학교 도서관"
  showDate: boolean;   // 촬영 날짜 자동 삽입
  position: "top-left" | "top-right" | "bottom-left" | "bottom-right";
  fontSize: number;    // 기본 16
  color: string;       // 기본 "#ffffff"
  opacity: number;     // 0~1, 기본 0.8
}
```

### 표시 예시
```
┌─────────────────────┐
│                     │
│    [합성된 사진]     │
│                     │
│         2026.04.08  │
│   ○○초등학교 도서관  │
└─────────────────────┘
```

### 구현 위치
- 관리자 페이지 (`/admin`): 워터마크 설정 UI 추가
- 촬영 캡처 시점: Canvas에 텍스트 그리기
- 저장소: `localStorage`에 워터마크 설정 저장

### 구현 순서
1. 워터마크 설정 데이터 구조 & localStorage CRUD
2. 관리자 페이지에 워터마크 설정 섹션 추가
3. 캡처 시 Canvas에 워터마크 렌더링 함수 구현
4. 실시간 미리보기에도 워터마크 표시 (선택)

## 2. 촬영 가이드 오버레이

### 목적
크로마키 영역에 "여기에 얼굴을 맞춰주세요" 가이드를 표시하여 초등학생이 쉽게 위치 잡기

### 구현 방식
- 촬영 화면에서 마스크 바운딩박스 기반으로 가이드 오버레이 표시
- 셔터 누르기 전까지만 표시, 촬영 시 자동 숨김

### 가이드 요소
```
┌─────────────────────┐
│                     │
│   ┌ ─ ─ ─ ─ ─ ┐   │
│   │   👤        │   │
│   │ 여기에 얼굴을│   │
│   │ 맞춰주세요! │   │
│   └ ─ ─ ─ ─ ─ ┘   │
│                     │
│     [ 📸 셔터 ]     │
└─────────────────────┘
```

### 디자인
- 마스크 바운딩박스에 **둥근 점선** 테두리
- 중앙에 반투명 얼굴 아이콘 + 텍스트
- 부드러운 펄스 애니메이션 (주목 유도)
- 카메라 영상이 표시되기 시작하면 텍스트만 숨기고 점선 유지

### 구현 순서
1. 마스크 바운딩박스 좌표를 CSS 좌표로 변환
2. 가이드 오버레이 컴포넌트 생성
3. 촬영 화면에 오버레이 배치
4. 카운트다운 시작 시 가이드 숨김

## 3. 스티커/꾸미기

### 목적
촬영된 사진에 별, 하트, 텍스트 등 스티커를 추가하여 재미 요소 강화

### 구현 방식
- 결과 화면에서 사진 위에 드래그 앤 드롭으로 스티커 배치
- Canvas로 최종 합성 후 PNG 다운로드

### 스티커 종류
```
이모지 스티커: ⭐ ❤️ 🌈 🎉 🦋 🌸 🎀 👑 🔥 ✨
텍스트 스티커: 사용자 입력 텍스트 (글꼴, 색상 선택)
프레임 스티커: 귀여운 테두리 프레임
```

### 스티커 조작
```typescript
interface Sticker {
  id: string;
  type: "emoji" | "text";
  content: string;       // 이모지 또는 텍스트
  x: number;             // 위치
  y: number;
  scale: number;         // 크기 (핀치 줌)
  rotation: number;      // 회전 (두 손가락 회전)
  color?: string;        // 텍스트 색상
}
```

### UI 플로우
```
결과 화면
  → [꾸미기] 버튼 클릭
  → 하단에 스티커 팔레트 표시
  → 스티커 탭 → 사진 위에 배치
  → 드래그로 이동, 핀치로 크기, 회전
  → [완료] → Canvas 합성 → 저장 가능
```

### 구현 순서
1. 스티커 데이터 구조 & 상태 관리 (Zustand)
2. 스티커 팔레트 UI (이모지/텍스트 탭)
3. 드래그 앤 드롭 배치 (터치 이벤트)
4. 핀치 줌 & 회전
5. Canvas 합성 렌더링
6. 결과 화면에 [꾸미기] 버튼 연결

## 4. 애니메이션 GIF

### 목적
촬영 전후 몇 프레임을 캡처하여 움직이는 GIF로 저장

### 구현 방식
- 셔터 전후 약 1~2초간 프레임을 캡처 (10~20fps)
- `gif.js` 라이브러리로 GIF 인코딩
- PNG 저장과 함께 GIF 저장 옵션 제공

### 캡처 타이밍
```
카운트다운 3, 2, 1 ...
                    ↓ 셔터 순간
[───프레임 캡처───]──[──프레임 캡처──]
   촬영 전 0.5초       촬영 후 0.5초
   약 10프레임          약 10프레임
           → 총 20프레임 GIF
```

### 기술 스택
```bash
npm install gif.js  # GIF 인코딩 라이브러리
```

### GIF 설정
```typescript
interface GifConfig {
  fps: number;        // 기본 10
  quality: number;    // 1(최고)~30(최저), 기본 10
  width: number;      // 리사이즈 (모바일 성능)
  height: number;
  duration: number;   // 전후 캡처 시간 (ms)
}
```

### 구현 순서
1. gif.js 라이브러리 설치 & 설정
2. 프레임 캡처 버퍼 구현 (링 버퍼, 최근 N프레임 유지)
3. 셔터 시점 전후 프레임 수집
4. GIF 인코딩 (Web Worker로 백그라운드 처리)
5. 결과 화면에 [GIF 저장] 버튼 추가
6. GIF 다운로드 / 공유

## 5. 책표지 내보내기/가져오기

### 목적
로컬 저장소 한계를 보완하여 데이터 백업 & 복원 지원

### 구현 방식
- 등록된 책표지 데이터를 JSON 파일로 내보내기
- JSON 파일을 가져와서 IndexedDB에 복원
- 이미지 데이터는 Base64로 JSON에 포함

### 내보내기 포맷
```json
{
  "version": 1,
  "exportedAt": "2026-04-08T12:00:00Z",
  "covers": [
    {
      "id": "uuid",
      "name": "어린왕자",
      "imageData": "data:image/png;base64,...",
      "maskData": "data:image/png;base64,...",
      "previewData": "data:image/png;base64,...",
      "createdAt": 1712563200000
    }
  ]
}
```

### UI
```
관리자 페이지
  ┌──────────────────────────┐
  │ [📥 내보내기] [📤 가져오기] │
  └──────────────────────────┘
  
  내보내기 → JSON 파일 다운로드
  가져오기 → 파일 선택 → 미리보기 → 확인 → 복원
```

### 주의사항
- 가져오기 시 기존 데이터와 ID 충돌 방지 (새 ID 발급 옵션)
- 파일 크기 경고 (이미지 포함 시 수십 MB 가능)
- 가져오기 전 미리보기로 확인

### 구현 순서
1. 내보내기: IndexedDB → JSON → 파일 다운로드
2. 가져오기: 파일 읽기 → JSON 파싱 → 유효성 검증
3. 가져오기 미리보기 모달
4. IndexedDB에 복원 (중복 처리)
5. 관리자 페이지에 내보내기/가져오기 버튼 추가

## 6. 프린터 연동

### 목적
촬영된 사진을 연결된 프린터(USB/WiFi)로 바로 출력

### 구현 방식
- 브라우저의 `window.print()` API 활용
- 인쇄 전용 레이아웃을 별도 생성하여 깔끔한 출력

### 인쇄 레이아웃
```
┌─────────────────────┐
│                     │
│    [촬영된 사진]     │
│                     │
│   2026.04.08        │
│   ○○초등학교 도서관  │
│                     │
│   Book Photo Booth  │
└─────────────────────┘

용지: 4x6 (포토 용지) 또는 A4
```

### 기술 접근
```css
/* 인쇄 전용 스타일 */
@media print {
  body * { visibility: hidden; }
  .print-area { visibility: visible; }
  @page { size: 4in 6in; margin: 0; }
}
```

### 고급 옵션
- 인쇄 품질을 위해 Canvas를 고해상도(300dpi)로 렌더링
- 포토 용지 크기 선택 (4x6, L사이즈, A4)
- 1장 / 2장 / 4장 레이아웃 선택

### 구현 순서
1. 인쇄 전용 CSS 미디어 쿼리 작성
2. 인쇄 레이아웃 컴포넌트 생성 (고해상도 Canvas)
3. 용지 크기 / 레이아웃 선택 UI
4. 결과 화면에 [인쇄] 버튼 추가
5. `window.print()` 호출
